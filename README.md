# 学校运动会管理系统 (C++)

本项目是一个基于C++实现的学校运动会管理系统，旨在提供运动会相关的各项管理功能。

## 项目目标
- 实现参赛单位、运动员、比赛项目的管理。
- 实现运动员报名登记，并满足相关限制条件。
- 实现计分规则的灵活设置。
- 自动生成秩序册（高级功能，逐步完善）。
- 实现比赛成绩的录入、查询、统计和汇总。
- 提供数据备份与恢复功能。

## 当前阶段

项目处于初期开发阶段，已完成核心数据结构和部分基础功能的框架搭建。

## 具体内容
【问题描述】

### 背景描述
“学校运动会管理系统”旨在简化运动会组织与管理工作。设计目标是支持丰富的管理功能，包括单位参赛、比赛项目设置、运动员信息管理、以及成绩统计与智能生成秩序册。

### 系统功能要求
1. **参赛单位与项目**
    - 系统支持**n个单位**（例如学院、系、年级等）参赛，单位名称需要可自定义；
    - 系统提供**m个男子项目**和**w个女子项目**，参赛单位可自由选择参赛项目；
    - 运动员每人最多只能报名**3项单项比赛**。

2. **比赛名次与计分**
    - 系统支持灵活设置计分规则，当前默认规则如下：
        - 若参赛人数 > 6 人，取前 5 名：
            - 第一名 7 分，第二名 5 分，第三名 3 分，第四名 2 分，第五名 1 分；
        - 若参赛人数 ≤ 6 人，取前 3 名：
            - 第一名 5 分，第二名 3 分，第三名 2 分；
        - 若参赛人数 < 4 人，取消该比赛项目；
    - 系统需根据运动会环境支持自定义计分规则配置。

3. **系统设置**
    - 提供以下可配置内容：
        - **参赛单位名称**；
        - **比赛项目设置**（支持田赛、径赛等条件分类设置，包括项目成绩排序规则）；
        - **运动员参赛项目限制**；
        - **比赛成绩名次与计分规则设置**。

---

【基本功能需求】

### 系统功能模块设计

1. **参赛项目发布**
    - 发布比赛项目信息，包括田赛与竞赛类别、参赛限制条件等。

2. **运动员报名登记**
    - 支持运动员报名功能，限制每位运动员最多参加 3 项单项比赛；
    - 自动判定报名信息中的冲突部分，并提供修正建议。

3. **参赛信息查询**
    - 查询参赛单位信息，包括单位详情和报名人数；
    - 查询比赛项目信息，包括参赛队伍和报名规则。

4. **秩序册自动生成**
    - 采用**智能算法**生成秩序册，优先级规则包括：
        - 避免**场地冲突**；
        - 同一运动员不同项目时间不应冲突；
        - 同一运动员尽量避免连续参赛；
        - 运动会总体时间优化（应尽量短）；
        - 平衡上午和下午比赛项目数量。

5. **比赛成绩查询**
    - 提供以下维度的比赛成绩查询：
        - 比赛项目维度；
        - 参赛单位维度；
        - 运动员维度；
    - 成绩展示示例：
      ```
      100米：
         第一名  张三    11＇＇2
         第二名  李四    11＇＇7
         第三名  王五    11＇＇9
         第四名  刘六    12＇＇5
         第五名  赵七    12＇＇8
         第六名  周八    13＇＇1
  
      跳高：
         第一名  张三    2米02
         第二名  李四    2米
         第三名  王五    1米90
         第四名  刘六    1米88
         第五名  赵七    1米82
         第六名  周八    1米80
      ```

6. **比赛成绩统计与汇总**
    - 按单位和运动员统计比赛成绩分数；
    - 自动生成各学院得分汇总与排名。

7. **数据备份与恢复**
    - 提供专项数据备份功能，便于数据恢复与导入。

---

【备注】
1. **秩序册自动生成**需使用较为智能的算法优化比赛安排，具体实现基于复杂规则匹配；
2. 在**参赛信息查询**模块中，可引入多种算法并优化性能。
