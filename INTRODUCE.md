# SchoolSports-Cpp 项目技术文档

## 1. 项目概述

SchoolSports-Cpp 是一个用C++开发的学校运动会管理系统，能够全面支持学校运动会的赛前准备、报名管理、赛程安排、成绩记录和统计分析等全流程。系统采用面向对象设计，实现了高内聚低耦合的模块化架构，支持运动会全生命周期管理。

## 2. 系统架构

### 2.1 总体架构

系统采用经典的MVC（Model-View-Controller）设计模式，分为三层：

```
┌───────────────────┐    ┌───────────────────┐    ┌───────────────────┐
│      UI 层        │◄───┤    控制器层       │◄───┤     组件层        │
│  (Views/输入输出) │    │  (Controllers)    │    │   (Models/数据)   │
└───────────────────┘    └───────────────────┘    └───────────────────┘
```

- **组件层（Components）**：核心数据模型和业务逻辑，包括单位、运动员、比赛项目、成绩等实体类及其管理类
- **控制器层（Controllers）**：处理用户操作，协调组件层和UI层的交互
- **UI层（UI）**：负责与用户交互，显示信息和获取输入


## 3. 核心组件详解

### 3.1 组件层（Components）

#### 3.1.1 Core 核心数据模型

**SystemSettings（系统配置）**
作为整个系统的配置中心，管理全局状态和设置。

**主要功能**：
- 工作流程阶段管理（项目设置、运动员报名、比赛进行）
- 全局设置参数管理（运动员最大参赛项目数、赛程锁定状态）
- 作为中央存储库管理所有实体对象的集合

**Unit（参赛单位）**
表示参与运动会的组织单位（如学院、班级）。

**主要属性**：
- ID、名称
- 所属运动员列表
- 总积分

**Athlete（运动员）**
表示参加比赛的个体运动员。

**主要属性**：
- ID、姓名、性别
- 所属单位ID
- 已报名项目列表

**CompetitionEvent（比赛项目）**
定义单个比赛项目的特征。

**主要属性**：
- ID、名称、项目类型（径赛/田赛）
- 性别要求
- 参赛运动员列表
- 计分规则ID
- 取消状态
- 场地、持续时间、开始和结束时间

**ScoreRule（计分规则）**
定义比赛项目的积分计算规则，支持复合规则模式。

**主要功能**：
- 基于参赛人数确定录取名次
- 各名次对应的得分计算
- 支持复合规则（不同参赛人数使用不同得分标准）

**Result（比赛成绩）**
管理单个运动员的比赛成绩记录。

**EventResults（项目成绩集）**
管理整个项目的成绩集合和积分计算。

**主要功能**：
- 记录运动员在特定项目中的名次和原始成绩
- 计算并分配积分
- 汇总项目整体成绩

**Schedule（赛程）**
负责比赛赛程的安排和管理。

**主要功能**：
- 自动生成比赛日程安排
- 管理项目时间分配
- 生成秩序册
- 检查时间和场地冲突

**Registration（报名管理）**
管理运动员报名相关的功能。

**主要功能**：
- 处理运动员报名和取消
- 项目报名人数检查
- 检查并标记人数不足的项目

**Workflow（工作流程）**
定义系统工作的不同阶段：
- SETUP_EVENTS（项目设置阶段）
- REGISTRATION_OPEN（运动员报名阶段）
- COMPETITION_RUNNING（比赛进行阶段）

**AutoTest（自动测试）**
提供系统自动化测试功能。

**Constants（常量定义）**
系统中使用的全局常量和配置参数。

#### 3.1.2 Manager 管理类

系统采用管理器模式对核心数据模型进行封装管理，主要包括：

- **UnitManager**: 参赛单位管理
- **AthleteManager**: 运动员管理
- **CompetitionEventManager**: 比赛项目管理
- **ScoreRuleManager**: 计分规则管理
- **EventResultsManager**: 成绩管理
- **WorkflowManager**: 工作流程阶段管理
- **SessionManager**: 时间段管理
- **VenueManager**: 场地管理
- **ScheduleLockManager**: 赛程锁定管理
- **SystemArgsManager**: 系统参数管理
- **DataManager**: 数据持久化管理
- **DataFileManager**: 文件操作管理

### 3.2 控制器层（Controllers）

#### 3.2.1 SystemSettingsController

系统设置和基础数据管理的控制器。

**主要功能**：
- 单位管理（添加、查看）
- 比赛项目管理（添加、查看）
- 计分规则管理（添加、编辑复合规则）
- 场地管理
- 时间段设置

#### 3.2.2 RegistrationController

运动员报名相关的控制器。

**主要功能**：
- 处理运动员报名和取消
- 显示报名状态和统计
- 检查并取消人数不足项目

#### 3.2.3 ScheduleController

赛程管理的控制器。

**主要功能**：
- 生成与管理赛程
- 锁定和解锁赛程
- 查看和验证秩序册

#### 3.2.4 ResultsController

成绩管理的控制器。

**主要功能**：
- 录入和修改比赛成绩
- 查看指定项目成绩
- 成绩自动排名和积分计算

#### 3.2.5 DataManagementController

数据管理的控制器。

**主要功能**：
- 数据备份
- 数据恢复
- 分阶段导入示例数据

#### 3.2.6 QueryController

提供系统数据查询功能。

**主要功能**：
- 提供各类数据查询接口
- 支持多条件查询和过滤

### 3.3 UI层（UI）

#### 3.3.1 UIManager

统一的UI管理类，处理所有用户交互。

**主要功能**：
- 输入输出工具（获取整数、字符串、浮点数输入）
- 菜单显示（不同阶段显示不同菜单）
- 数据列表展示（单位、运动员、比赛项目、成绩等）
- 消息和错误提示

## 4. 工作流程与阶段管理

系统实现了三阶段工作流管理，明确区分运动会的不同准备和执行阶段：

### 4.1 阶段1：项目设置阶段（SETUP_EVENTS）

**主要功能**：
- 设置参赛单位
- 创建比赛项目
- 配置场地信息
- 设置时间段
- 管理计分规则
- 生成初步赛程

### 4.2 阶段2：运动员报名阶段（REGISTRATION_OPEN）

**主要功能**：
- 添加运动员信息
- 报名参赛项目
- 查看报名情况
- 检查项目报名人数
- 取消报名不足项目
- 生成最终赛程

### 4.3 阶段3：比赛进行阶段（COMPETITION_RUNNING）

**主要功能**：
- 录入比赛成绩
- 查看各项目成绩
- 查看单位总分排名
- 数据备份与统计

## 5. 核心功能亮点

### 5.1 复合计分规则设计

实现灵活的计分规则模式，可以根据参赛人数自动应用不同的计分标准：
- 参赛人数超过6人：录取前5名，得分为7-5-3-2-1分
- 参赛人数为4-6人：录取前3名，得分为5-3-2分
- 参赛人数不足4人：项目取消

### 5.2 智能赛程安排

根据项目特性、场地限制和时间窗口，自动生成合理的赛程安排：
- 自动避免场地冲突
- 合理安排上午/下午时段
- 保证运动员参赛间隔充足

### 5.3 分阶段示例数据导入

提供按阶段导入示例数据的功能，帮助快速测试和演示系统：
- 阶段1：导入单位、场地和比赛项目
- 阶段2：导入运动员及其报名信息
- 阶段3：导入比赛成绩

### 5.4 阶段性菜单适配

根据当前工作流阶段自动调整菜单选项和功能权限：
- 不同阶段显示不同系统设置选项
- 锁定状态下限制某些修改操作
- 清晰标识当前系统所处阶段

## 6. 技术实现细节

### 6.1 代理模式的应用

系统使用代理模式将核心数据模型的管理封装到对应的Manager类中，通过SystemSettings类提供统一访问接口：

```cpp
// SystemSettings类中包含多个Manager类实例
class SystemSettings {
public:
    // 代理类实例
    UnitManager units;
    AthleteManager athletes;
    CompetitionEventManager events;
    ScoreRuleManager rules;
    EventResultsManager results;
    WorkflowManager workflow;
    SessionManager sessions;
    VenueManager venues;
    ScheduleLockManager schedule;
    SystemArgsManager args;
    
    // ...其他成员和方法
};
```

### 6.2 C++20特性的应用

项目使用了C++20标准，利用了多项现代C++特性：
- std::ranges用于集合操作和处理
- std::optional用于可选值处理
- 利用现代C++的类型推导和智能指针

### 6.3 构建系统

项目使用CMake作为构建系统，支持跨平台构建：
- 设置C++20标准要求
- 支持构建命令导出，方便IDE集成
- 清晰组织项目文件结构

## 8. 使用指南

### 8.1 编译与运行

假设使用 g++ 编译器：

```bash
# 使用CMake构建
mkdir build && cd build
cmake ..
make

# 运行程序
./SchoolSports-Cpp
```

也可以直接编译：

```bash
g++ -std=c++20 -Wall -Wextra src/*.cpp -Iinclude -o sports_meet_system
```

### 8.2 使用流程

1. 初始设置阶段：添加单位、创建比赛项目、设置计分规则
2. 报名阶段：添加运动员信息、进行项目报名
3. 比赛阶段：录入成绩、查看统计和排名结果

### 8.3 示例数据

系统提供了分阶段导入示例数据的功能，可以通过主菜单中的"导入示例数据"选项快速填充系统进行测试。
